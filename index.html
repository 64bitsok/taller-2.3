VISTAS EN
ORACLE SQL

// Richard Triveños Llaccta · Grupo 64bits · 2026

Una vista (VIEW) es un objeto virtual en la base de datos que almacena una consulta SQL compleja como si fuera una tabla. No guarda datos físicos — solo guarda la lógica de la consulta. Cada vez que se consulta la vista, Oracle ejecuta esa lógica en tiempo real, devolviendo los datos actualizados.

Sintaxis Fundamental

oracle_view.sql
-- Creación básica de una vista CREATE OR REPLACE VIEW v_NombreVista AS SELECT t1.campo1, t1.campo2, t2.campo3 FROM tabla1 t1 JOIN tabla2 t2 ON t1.id = t2.id_fk; -- Uso: exactamente igual que una tabla SELECT * FROM v_NombreVista; SELECT COUNT(*) FROM v_NombreVista WHERE campo1 = 'valor';

Vista Principal — v_PacienteEnfermedad

v_PacienteEnfermedad

Unifica 4 tablas (tPaciente, tDiagnostico, tEnfermedadEnElDiagnostico, tEnfermedad) en un solo objeto consultable. En lugar de escribir 4 JOINs cada vez, basta con SELECT * FROM v_PacienteEnfermedad.

v_PacienteEnfermedad
CREATE OR REPLACE VIEW v_pacienteEnfermedad AS SELECT p.codP AS codigoP, p.nombresP, p.paternoP, p.maternoP, d.codD AS codigoD, d.descripcionD AS nombreD, e.nombreE FROM tPaciente p JOIN tDiagnostico d ON p.codP = d.codP JOIN tEnfermedadEnElDiagnostico ed ON d.codD = ed.codD JOIN tEnfermedad e ON ed.codE = e.codE; -- Contar enfermedades por paciente usando la vista: SELECT COUNT(*) AS cantidad_de_enfermedades, nombresP, paternoP, maternoP FROM v_pacienteEnfermedad GROUP BY nombresP, paternoP, maternoP;

¿Por qué usar Vistas?

✗ Sin vistas — consulta directa

  • Escribir JOINs complejos cada vez
  • Código repetido en múltiples lugares
  • Expone todas las columnas (seguridad)
  • Mantenimiento costoso al cambiar tablas
  • Difícil de leer y auditar

✓ Con vistas — acceso simplificado

  • SELECT simple sobre un objeto legible
  • Lógica centralizada en un solo lugar
  • Control de columnas visibles (seguridad)
  • Actualiza automáticamente al cambiar tablas
  • Código limpio y reutilizable

Tipos de consultas sobre Vistas

📋

Consulta simple

SELECT * FROM v_vista — accede como tabla normal, devuelve todos los registros de la consulta almacenada.

📊

Agregación

COUNT(), SUM(), AVG() con GROUP BY sobre la vista. Ideal para reportes estadísticos sin reescribir JOINs.

🔍

Filtros con HAVING

Filtra grupos después del GROUP BY. Útil para "médicos con 2+ especialidades" o umbrales de conteo.

🚫

NOT IN / NOT EXISTS

Detecta registros sin relación: enfermedades no diagnosticadas, especialidades sin médico asignado.

🔗

Vista sobre vista

Una consulta puede usar otra vista como fuente. Ej: filtrar v_MedicosEspecialidades con HAVING.

🔄

DISTINCT

Elimina duplicados en vistas con múltiples JOINs. Esencial cuando un paciente tiene varios diagnósticos.

Conclusión

La implementación de vistas en el sistema clínico permite transformar una estructura de datos compleja y fragmentada en información simplificada, segura y accesible. Las vistas facilitan la generación de reportes estratégicos — como especialidades desatendidas o el conteo de enfermedades por paciente — sin necesidad de reescribir consultas extensas ni comprometer la integridad de las tablas originales.
